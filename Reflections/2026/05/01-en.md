## Analysis of “Unable to store files” Issue in TIM 4.0.98 on Android 16

### Problem

TIM 4.0.98 reports “Unable to store files, please check SD card status” after launching on Android 16 (crDroid), and fails to load images and cache files.

### Root Cause

There is a bug in Android 16 where the directory `/storage/emulated/0/Android/data/<package_name>/` is not automatically created after app installation. TIM’s NTKernel (C++ layer) directly constructs file paths and writes files without going through the Java API `getExternalFilesDir()`. As a result, the system does not trigger automatic directory creation, causing all file write operations to return EIO (error -5).

### Key Logs

```
W ContextImpl: Failed to ensure /storage/emulated/0/Android/data/com.tencent.tim/cache
E NTKernel: CreateTempDir Fail. error:-5 path=/storage/emulated/0/Android/data/com.tencent.tim/Tencent/Tim/chatpic/Temp
E NTKernel: CreateFileDir Fail. error:-5 path=/storage/emulated/0/Android/data/com.tencent.tim/Tencent/Tim/chatpic/chatraw/a29
E NTKernel: fail to create directory:/storage/emulated/0/Android/data/com.tencent.tim/Tencent/Tim/chatpic/Temp(-5)
```

### Temporary Solution

Get TIM’s uid:

```bash
su -c "stat -c %u /data/data/com.tencent.tim"
```

Manually create directories and set correct ownership (replace `10499` with actual uid):

```bash
su -c "mkdir -p /storage/emulated/0/Android/data/com.tencent.tim/cache && \
mkdir -p /storage/emulated/0/Android/data/com.tencent.tim/files && \
chown -R 10499:10499 /storage/emulated/0/Android/data/com.tencent.tim && \
chmod -R 771 /storage/emulated/0/Android/data/com.tencent.tim"
```

### Notes

* This issue is an upstream Android 16 bug, not specific to crDroid; it can also be reproduced on stock Pixel devices
* Apps that create directories via Java API (`getExternalFilesDir()`) are not affected
* TIM has not adapted to Android 16, and its testing scope is mainly limited to domestic OEM ROMs
