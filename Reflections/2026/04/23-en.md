## I. Fixing SSH Key Management in the Native Environment
I resolved the technical hurdles encountered when logging into a VPS from the local PowerShell using the `ssh` command:

1. **Resolved the `ssh-agent` Connection Failure Issue**
To address the `Error connecting to agent` message that appeared when executing `ssh-add`, I ran the following commands with administrator privileges to set the `ssh-agent` service to start automatically and then enabled it:

- Configure SSH settings
```powershell
Set-Service -Name ssh-agent -StartupType Automatic
Start-Service ssh-agent
```
```.config
Host "VPS Domain"
Hostname "VPS IP"
User root
Port 22
IdentityFile  path\to\id_ed25519
```

2. **Eliminated PowerShell Startup Lag (1s+)**
To resolve the delay caused by Conda initialization and online update checks occurring every time the Shell was opened, I configured Conda within my `$PROFILE` to load on demand via a `function cn`, and added the line `$env:POWERSHELL_UPDATECHECK = 'Off'`.

## II. Using Specialized SSH Connection Tools
To facilitate easier connection and management of the VPS, I configured and tested several specialized SSH tools:

* **WindTerm**: I utilized this high-performance SSH client, noting its exceptionally fast real-time response speed.
* **MobaXterm**: By copying the localization plugin files from the Chinese-localized portable version (`MobaXterm_Portable_v24.3_chs`) into the installation directory of the official installed version, I successfully enabled Chinese localization for the official release. I leveraged its integrated SFTP transfer capabilities and Chinese interface to manage files on the remote VPS.

## III. Eclipse Plugin Installation and Localization
I attempted to localize the user interface of Eclipse:
* **Installation Process**: I simply placed a custom localization plugin package directly into the `dropins` folder located within the Eclipse installation directory.
