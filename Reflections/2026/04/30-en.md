## Solving Git Proxy Issues in WSL

By design, WSL (Windows Subsystem for Linux) is a virtual machine with its own independent network stack. Setting it up to be transparently proxied by the host machine can be complex. Therefore, I chose a simpler approach: using the host machine's LAN IP address to set up a LAN proxy.

### 1. HTTP/HTTPS Proxy
```bash
git config --global http.proxy http://<Host_IP>:10808
git config --global https.proxy http://<Host_IP>:10808
```

### 2. SSH Proxy via HTTP
For SSH connections, you can use an HTTP proxy by leveraging **connect-proxy**:
```bash
apt install -y connect-proxy
```

Configure your `~/.ssh/config`:
```bash
Host github.com
    HostName ssh.github.com
    Port 443
    User git
    IdentityFile /root/.ssh/id_ed25519-git
    ProxyCommand connect-proxy -H <My_Local_IP>:<port> %h %p
```


## Optimizing PowerShell Startup Speed

Reduced PowerShell startup time from **1418ms** to nearly instantaneous.

**The Problem:**
The message "Loading personal and system profiles took 1418ms" appeared. Investigations showed that `conda.exe` was being called during initialization in `profile.ps1`, compounded by overhead from Windows Defender scans.

**The Solution:**

1. **Remove Conda Initialization:** Delete the Conda initialization block (approx. 5 lines from `#region` to `#endregion`) in `D:\UserName\Documents\PowerShell\profile.ps1`.

2. **Windows Defender Exclusions:** Run the following to exclude PowerShell processes and paths from real-time scanning:
   ```powershell
   Add-MpPreference -ExclusionProcess "pwsh.exe"
   Add-MpPreference -ExclusionPath "D:\Software\PowerShell\7"
   Add-MpPreference -ExclusionPath "D:\UserName\Documents\PowerShell"
   ```

3. **Disable Update Check:**
   ```powershell
   [System.Environment]::SetEnvironmentVariable('POWERSHELL_UPDATECHECK', 'Off', 'User')
   ```

4. **On-Demand Activation:** Move the Conda activation logic to a function (e.g., `cn`) within `Microsoft.PowerShell_profile.ps1`. Type `cn` only when you need to use Conda.

### Final Configuration

```ps1
# 1. Force disable PowerShell update check at startup (saves hundreds of ms)
$env:POWERSHELL_UPDATECHECK = 'Off'

# 2. Define on-demand Conda activation function
# Type 'cn' in the terminal to activate Conda. This avoids loading it on every startup.
function cn {
    $Env:CONDA_EXE = "D:\Software\Anaconda3\Scripts\conda.exe"
    $Env:_CE_M = ""
    $Env:_CE_CONDA = ""
    $Env:_CONDA_ROOT = "D:\Software\Anaconda3"
    $Env:_CONDA_EXE = "D:\Software\Anaconda3\Scripts\conda.exe"
    $CondaModuleArgs = @{ChangePs1 = $True}

    # The heavy module is only imported when 'cn' is executed
    Import-Module "$Env:_CONDA_ROOT\shell\condabin\Conda.psm1" -ArgumentList $CondaModuleArgs
    Remove-Variable CondaModuleArgs

    Write-Host "--- Conda Environment Activated ---" -ForegroundColor Green
}
```

## Resolving Git "Modified" Status caused by Line Endings (LF vs. CRLF)

To handle inconsistencies between Linux and Windows line endings, create a `.gitattributes` file.

### 1. Create `.gitattributes`

```.gitattributes
* text=auto

# Force LF for source files and documentation
*.cpp text eol=lf
*.h text eol=lf
*.c text eol=lf
*.hpp text eol=lf
*.txt text eol=lf
*.md text eol=lf
*.json text eol=lf
*.xml text eol=lf
*.yml text eol=lf
*.yaml text eol=lf
*.sh text eol=lf
Makefile text eol=lf

# Keep CRLF for Windows-specific files
*.sln text eol=crlf
*.vcxproj text eol=crlf
*.vcxproj.filters text eol=crlf
*.vcxproj.user text eol=crlf

# Mark binaries
*.png binary
*.jpg binary
*.jpeg binary
*.gif binary
*.ico binary
*.exe binary
*.dll binary
*.lib binary
```

### 2. Apply Changes
Run the following commands to normalize the repository:
```bash
git add --renormalize .
git commit -m "build: force all text files to LF for cross-platform consistency"
```
