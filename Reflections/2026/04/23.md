# 2026-04-23 解决了通过 SSH 连接 VPS 的一系列环境障碍

[返回上一页](./../04.md)

## 一、 修复原生环境的 SSH 密钥管理
我解决了在本地 PowerShell 通过 `ssh` 命令登录 VPS 时的技术阻碍：

1. **解决 ssh-agent 无法连接问题**
   针对执行 `ssh-add` 报错 `Error connecting to agent` 的问题，我通过管理员权限执行以下命令，将 `ssh-agent` 服务设为自动启动并开启：
   
   - 设置ssh config
   ```powershell
   Set-Service -Name ssh-agent -StartupType Automatic
   Start-Service ssh-agent
   ```
   ```.config
   Host "VPS Doamin"
	Hostname "VPS IP"
	User root
	Port 22
	IdentityFile  path\to\id_ed25519
    ```

2. **消除 PowerShell 启动卡顿 (1s+)**
   针对每次打开 Shell 都要初始化 Conda 和联网检查更新导致的延迟，我在 `$PROFILE` 中将 Conda 设置为 `function cn` 按需加载，并添加了 `$env:POWERSHELL_UPDATECHECK = 'Off'`。

## 二、 使用专业的 SSH 连接工具
为了更方便地连接和管理 VPS，我配置并测试了专门的 SSH 工具：

* **WindTerm**：使用了这款高性能的 SSH 客户端，其实时响应速度非常快。
* **MobaXterm**：通过将 `MobaXterm_Portable_v24.3_chs` 汉化版的插件覆盖移动到官方安装版的文件中，实现了官方安装版的汉化使用。利用其集成的 SFTP 传输和中文界面来管理远程 VPS 文件。

## 三、 Eclipse 插件安装与汉化
我尝试了对 Eclipse 的界面汉化：
* **安装操作**：直接在 Eclipse 安装目录的 `dropins` 文件夹下放置了自定义汉化插件包。
* **实际效果**：成功实现了界面汉化，但发现部分深层菜单和设置项的翻译并不完全。
